## 1. 产品简介

「班级宠物园」是一款面向课堂管理的轻量工具：通过 **加分/扣分评价** 推动学生的正向行为，同时将积分与“宠物成长/徽章/小商店兑换”等机制结合，提升参与度与持续性。

本软件默认 **无需登录**，数据保存在本机浏览器（`localStorage`）。同时支持手动导入导出、记录导出、以及自动备份到本地文件（浏览器支持时）。

---

## 2. 快速开始（3 分钟上手）

1. 打开网页后，先确认右上角班级名称（点击可进入 **班级与学生管理**）。
2. 在 **班级与学生管理** 中：
   - 新增班级（可选）
   - 添加学生或批量导入学生名单
3. 回到主界面：
   - 未领养的学生卡片显示 🥚，点击卡片或“领养宠物”按钮为学生选择宠物
4. 给学生评价：
   - 点击已领养学生卡片 → 选择加分/扣分规则 → 完成一次评价

---

## 3. 主界面说明

### 3.1 顶部功能区
- **搜索学生**：按姓名或学号筛选
- **排行榜**：查看徽章/等级等排序
- **小商店**：设置商品与兑换
- **评价记录**：查看最近 50 条，并支持导出
- **撤回评价**：撤回最近一次评价（有条件限制，见下文）
- **批量评价**：进入批量选择模式（对多名学生同时评价）
- **设置与帮助**：规则管理、成长设置、数据管理等

### 3.2 学生卡片
- 未领养：显示 🥚，可点击领养
- 已领养：
  - 显示宠物与等级（Lv.）
  - 显示积分与徽章
  - **点击卡片直接进入加分/扣分**
  - 左上角「⟳」支持更换宠物（更换后会清空该学生积分）

## 4. 班级与学生管理

入口：点击顶部班级名或在设置中进入。

### 4.1 班级列表
- 支持新增/编辑/删除班级
- 当前使用的班级会标记“当前使用”
- **编辑/删除按钮为悬浮显示（hover）**

### 4.2 学生列表
- 支持新增学生（姓名必填，学号可选且需为纯数字）
- 支持批量导入（按提示格式：`姓名 学号`，学号可省略）
- 支持编辑/删除学生
- **编辑/删除按钮为悬浮显示（hover）**

---

## 5. 领养与更换宠物

入口：
- 学生未领养：点击学生卡片或“领养宠物”
- 学生已领养：点击卡片左上「⟳」进行更换

说明：
- 支持批量领养（批量模式下）
- **更换宠物会清空该学生积分**，以保证激励公平性

---

## 6. 加分/扣分评价（规则选择）

入口：点击已领养的学生卡片（或批量模式）。

流程：
1. 选择分类（学习/行为/健康/其他）
2. 点击某条规则卡片，即对学生加分/扣分

说明：
- 规则可配置（见“加减分规则管理”）
- 系统会记录每次评价，支持在“评价记录”中查看

---

## 7. 加减分规则管理（设置 → 加减分规则）

页面结构：
- 左侧：分类目录 + 规则统计（加分/减分条数）
- 右侧：当前分类下的规则卡片列表

支持操作：
- **新增指标**：选择图标、填写名称、选择分类、范围（全部/本班）、评价导向（奖励/扣分）、分值
- **编辑/删除**：对已有规则进行修改或删除
- 规则卡片展示：
  - 图标
  - 名称
  - 分值（加分绿色、减分红色）
  - 生效范围标签（全部班级/本班）

---

## 8. 小商店（兑换）

入口：顶部「小商店」。

功能概览：
- 商品列表：展示可兑换商品（徽章作为货币）
- 记录：查看兑换记录
- 管理商品：新增/编辑/删除商品（含图标、价格、库存、启用状态等）

---

## 9. 排行榜

入口：顶部「排行榜」。

默认按：
1. 徽章数
2. 宠物等级
3. 积分

（具体排序以当前版本界面为准）

---

## 10. 评价记录（最近 50 条）

入口：顶部「评价记录」。

支持：
- 查看最近 50 条记录（学生、规则、分类、时间、分值）
- **撤回单条记录**（仅对支持撤回的记录可用）
- **导出 Excel**：当前实现为 CSV 文件（Excel 可直接打开，且带 BOM 解决中文乱码）

导出字段：
- 时间、学生、分类、规则、加/减、分值

---

## 11. 撤回评价

入口：
- 顶部「撤回评价」：撤回最近一条
- 评价记录中「撤回」：撤回指定记录（若该条记录支持撤回）

说明：
- 对旧版本历史数据，可能因缺少“操作前状态”而无法撤回

---

## 12. 数据管理（设置 → 数据管理）

### 12.1 导入/导出（JSON）
- **复制导出**：将当前数据 JSON 复制到剪贴板
- **下载导出**：下载 JSON 文件，便于备份/迁移
- **导入数据**：粘贴 JSON 导入（会覆盖当前数据）

### 12.2 自动备份到本地文件（可选）
说明：
- 基于浏览器的 File System Access API（部分浏览器支持，如 Chrome/Edge）
- 需要首次选择备份文件并授权
- 支持开启/关闭、立即备份、清除目标、设置备份间隔

注意：
- “清除目标”仅清除浏览器记住的备份文件，不会删除磁盘上的文件
- 浏览器安全限制：无法读取完整文件路径，通常只能显示文件名

## 13. 本地打包与部署（内网/公网）

### 13.1 打包

```bash
npm install
npm run build
```

生成目录：`dist/`

### 13.2 本地/内网访问
推荐用静态服务器提供 `dist/`（避免 `file://` 资源限制）：

```bash
npm run preview -- --host 0.0.0.0 --port 4173
```

内网访问：`http://内网IP:4173/`

## 14. 常见问题（FAQ）

### Q1：数据会丢吗？
默认数据在浏览器本地 `localStorage`。建议：
- 定期“下载导出 JSON”
- 开启“自动备份到本地文件”（浏览器支持时）

### Q2：为什么目标文件不显示路径？
浏览器出于安全限制，通常无法读取完整路径，只能读取文件名或“已选择/未选择”状态。

### Q3：能做账号登录、线上同步吗？
可以做后端版本，建议先做“云存储同步 MVP”，再迭代账号、兑换码、时长限制等商业化能力。
